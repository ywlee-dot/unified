## 전체 아키텍처

### 2개 파이프라인이 1개 워크플로우에 공존

```
[파이프라인 A — Webhook 연동용]
  Webhook → 데이터 입력 설정 → 페이지 설정1 → ... → 엑셀 → 결과 준비 → 파일 업로드 → 콜백 전송

[파이프라인 B — n8n 폼 직접 실행용] ← 이번 분석 대상
  사용자 입력 폼 → 데이터 입력 설정1 → 페이지 설정 → ... → 엑셀 파일 생성1
```

차이점:
- A: 백엔드에서 트리거, 결과를 콜백으로 반환 (파일 업로드 + 콜백 전송)
- B: n8n UI에서 직접 �� 입력, 엑셀 생성까지만 (콜백 없음)
- 핵심 로직(구글 검색 → AI 필터링 → 엑셀)은 동일, 노드가 접미사 "1"로 복제됨

### 파이프라인 B 전체 흐름

```

## Step 6: 구글 결과 파싱

### n8n 노드
- 타입: `n8n-nodes-base.code` (JavaScript)

### 코드 로직
```javascript
const allInputs = $input.all();  // 4개 페이지 결과 전부
const allItems = [];

for (const input of allInputs) {
  const serpResults = input.json;

  // SerpAPI 응답에서 뉴스 결과 필드 탐색 (우선순위)
  const items = serpResults.organic_results
    || serpResults.news_results
    || serpResults.top_stories
    || [];

  for (const item of items) {
    allItems.push({
      json: {
        title: item.title || '',
        link: item.link || item.url || '',
        snippet: item.snippet || item.description || '',
        date: item.date || '',
        source: '구글'
      }
    });
  }
}

return allItems;
```

### 수행 내용
1. 4개 페이지 응답을 하나로 합침
2. 각 응답에서 `news_results`, `organic_results`, `top_stories` 중 존재하는 필드 선택
3. 각 결과에서 title, link, snippet, date만 추출하여 표준화
4. source는 '구글'로 고정

### 입출력
- 입력: 4개 SerpAPI 응답 JSON
- 출력: 40개 표준화된 뉴스 아이템

### 시뮬레이션 결과
```

표준화 필드: title, link, snippet, date, source
```

## Step 7: 중복 제거

### n8n 노드
- 타입: `n8n-nodes-base.code` (JavaScript)

### 코드 로직
```javascript
const items = $input.all();
const seen = new Set();
const uniqueItems = [];

for (const item of items) {
  const data = item.json;
  
  // URL 또는 제목을 기준으로 중복 체크
  const link = data.link || '';
  const title = data.title || '';
  const identifier = link || title;
  
  if (identifier && !seen.has(identifier)) {
    seen.add(identifier);
    uniqueItems.push(item);
  }
}

return uniqueItems;
```

### 수행 내용
- URL이 있으면 URL로 중복 판별, 없으면 제목으로 판별
- Set 자료구조로 O(1) 중복 검사

### 입출력
- 입력: 40건
- 출력: 38건 (2건 중복 제거)

### 시뮬레이션 결과
```
전체 40건 중 38건 고유, 2건 중복 제거
```

## Step 10: JSON 파싱

### n8n 노드
- 타입: `n8n-nodes-base.code` (JavaScript)

### 코드 로직 (핵심부)
```javascript
// AI 응답에서 JSON 추출
function stripCodeFences(s) {
  return String(s || '')
    .replace(/```(?:json)?\s*/gi, '')
    .replace(/```/g, '')
    .trim();
}

function extractFirstJson(text) {
  // 첫 번째 [ 또는 { 를 찾아서 브레이스/브라켓 매칭으로 추출
  const s = String(text || '');
  const startArr = s.indexOf('[');
  const startObj = s.indexOf('{');
  // ... 균형 잡힌 괄호 매칭으로 첫 JSON 덩어리 추출
}

// 여러 아이템의 output을 전부 파싱해서 합치기
const allItems = $input.all();
const allParsed = [];

for (const item of allItems) {
  const raw = item.json.output || '';
  const cleaned = stripCodeFences(raw);
  const jsonStr = extractFirstJson(cleaned);
  
  if (jsonStr) {
    const parsed = JSON.parse(jsonStr);
    // 배열이면 펼치기, 객체면 단일 아이템
    if (Array.isArray(parsed)) {
      allParsed.push(...parsed);
    } else {
      allParsed.push(parsed);
    }
  }
}
```

### 수행 내용
1. AI Agent 응답의 `output` 필드에서 텍스트 추출
2. 마크다운 코드블록(```json```) 제거
3. 브레이스/브라켓 매칭으로 첫 번째 유효 JSON 추출
4. JSON.parse() 실행
5. 배열이면 펼치고, 객체면 단일 아이템으로 처리
6. 결과 0건이면 빈 아이템 1개 반환 (No output 방지)
